GDAL

KMSTrans2 / trui depends on GDAL/OGR for transformation of data sources in the batch tab as well as for displaying a coast line in the map window. Currently only the OGR part of GDAL is used to transform vector data.

Windows only:

A prepackaged version of KMSTrans2 with prebuilt binaries will come also with a (minimal) GDAL installation, which the binaries are linked against.
Relative to the root directory  (where the Trui.py or KMSTrans2.exe file is located) the structure of this installations is:

  • gdal/bin - a folder containing shared libraries (.dll), e.g. gdalxx.dll and various additional libraries that this dll is linked against.
  • gdal/plugins - a folder containing optional plugins for GDAL (might be empty or non existing, depending on the distribution).
  • gdal/gdal-data - a folder contaning data files needed by various GDAL drivers.

You can override the GDAL installation by simply dropping in another compatible GDAL installation with a similar directory structure.
 If you already have a compatible GDAL installation on your computer which is included in your PATH environment variable, you can simply delete the gdal folder in the root directory, or run the program from the command line with a -my_gdal option.

Building binaries yourself:

If you build the binaries yourself with the included build script - build.py - it is up to you to choose an appropriate GDAL installation/version to link against.
