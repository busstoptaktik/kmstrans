Batch Tab

FBtrans uses a command line program in the ./bin subfolder called trogr to perform transformations of datasources like files and databases.
trogr is linked against GDAL which must be installed on your system in order for the program to run.

Formats

Generally input and output datasource formats will be the same, except for input formats readable by OGR, where a specific OGR output format can be chosen. See a list of OGR formats here.

  • OGR - a range of common vector formats, including various databases, depending on your gdal installation.
Available input and output vector data formats will depend on your GDAL installation. OGR will try to open an input datasource by iteration over available drivers. This behavior can be overridden by specifying a specific input driver, e.g. (if the WFS driver is available):

WFS:http://kortforsyningen.kms.dk/service?servicename=dagi_gml2&service=WFS&version=1.0.0&login=usr_name&password=pass_wd

Refer to the list of OGR formats for more details.
  • TEXT - simple column based text format. Column separator and geometry columns can be specified in a settings dialog. The column separator defaults to all whitespace
e.g. a valid format could be:

K-47-200 ,  11.75 , 55.10 , 100 , windy
CITY,    9.31, 56.20 , 127, sunny

where col_x=2, col_y=3, col_z=4 and the column separator is ","  (comma).
An input system can be specified like using a hash in front of a minilabel, e.g.:   #geoEwgs84    (geographical coordinates in WGS84 datum, ellipsoidal heights).

NOTE: the TEXT driver expects geographical input coordinates to be in decimal degrees. Generally input coordinates should be decimal numbers without spaces or units. If you have a more complicated format consider using the KMS-format which accepts things like units and spaces in coordinates (and remember to use two spaces as column separator).

  • KMS - A special 'old' text format internal to KMS/GST. Column separator is *always* at least two spaces and  coordinates have units, e.g.:
BUDD   6 143 200.5 m  512 200.12 m  ...some other attributes...

So the format is:

<Station_name> <two_spaces> <X or Y>  <two_spaces> <X or Y> <two_spaces> <An optional Z-value - depending on the system> <two_spaces> <extra_attributes>

The order of the coordinates, e.g. X,Y (,Z) or Y,X (,Z) depends on the input system, which is specified like for the TEXT-format, e.g. #utm32_wgs84

  • DSFL - a specific danish exchange format for vector data.
